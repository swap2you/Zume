# Zume Agent Instructions

1. Keep the product local-first and CLI-based. Do not introduce a web application unless explicitly requested.
2. Treat candidate information as confidential. Never commit real candidate files or PII.
3. Use exact evidence from the resume or interviewer notes. Do not invent experience.
4. Generate DOCX outputs with readable spacing, color-coded sections, headers, footers and page numbers.
5. Every workflow must end in validation. Do not declare success when tests or document checks are skipped.
6. Preserve the exact trigger phrases defined in `config/triggers.yaml`.
7. Keep modules small, typed and testable.
8. Do not push changes unless the user explicitly authorizes it.
